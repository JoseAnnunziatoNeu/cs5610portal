Javascript

OVERVIEW: JavaScript is an interpreted language that became popular with Web developers for creating interactive, dynamic Web pages. It's popularity continues to grow as it finds its uses on the server side and database applications. This module focuses on JavaScript's role in the Web browser. Other modules will focus on other uses of JavaScript.

On the browser, JavaScript allows to create programs that run as part of a Web page. JavaScript can be either embedded within a Web page or downloaded as a separate JavaScript file referenced by the Web page. Either way the JavaScript code generally pertains to that one Web page. The intent is for the JavaScript code to take programmatic control of the Web page and respond to user gestures and dynamically manipulate the Web page by adding, removing, or changing the DOM of the Web page according to pre-programmed JavaScript code. For instance, JavaScript can be used to validate forms the user is filling out, giving immediate feedback whether the data is valid or not by highlighting form elements, showing or hiding icons or alert boxes.

Intro to JavaScript

JavaScript can be embedded in an HTML document by using the script element. Usually the script element goes in the head element, but it can appear anywhere in the HTML document and there can be any number of script elements. It is common to declare script elements at the end of the body element to improve SEO (Search Engine Optimization) performance, and page rendering performance. Within the script we can embed JavaScript code such as variables, arrays, objects, loops, branching, and functions.

Declare variables using the var keyword as shown below: 

var a = 2;
var b = 4;
var c = 2 + 4;
alert(c);

The JavaScript Document Object

When you type a URL in a browser address field, the browser makes an HTTP GET request to a server that's listening for these types of requests. Usually the request refers to or resolves to some HTML document. The server responds by streaming the document back to the browser. The browser downloads the document into its file cache and parses it into its memory. The document is parsed into an object representation of the document referred to as the DOM or Document Object Model. The DOM has an object representation of everything that is rendered on the page including the text, font, colors, layout, and white spaces in between. The DOM is the browser's interpretation of the plain text HTML document that is downloaded. Whereas the HTML might have syntax errors or missing elements, improper nesting, missing closing tags, the DOM is the interpretation of the text into a proper object representation of the page.

With JavaScript, it is possible to programmatically query, interact, and manipulate the DOM. Any modifications to the DOM are reflected onto the page. By changing the DOM, JavaScript can dynamically change what is displayed in the browser. The HTML page is available through JavaScript through an object called document. Through the document object we can use JavaScript to remove or add elements, change element's properties such as background color, font, padding and margins.

Getting Elements by Id

Use JavaScript's getElementById to get a hold of a particular HTML element identified by it's ID attribute. First use the ID attribute to uniquely identify an element. For instance the code below identifies a text field as the firstName text field: 

<input id="firstName"/>

The following code retrieves the text field by its ID and stores a reference to it in the firstNameField variable: 

var firstNameField = document.getElementById("firstName");

Once we have a reference to an element in the DOM, we can invoke various methods on them to apply style changes and behaviours. For instance, we can use the value attribute to set the text field's value. 

var firstNameField = document.getElementById("firstName");
firstNameField.value = "Alice";

JavaScript OnClick Event

HTML elements provide various attributes that allow binding user gestures to behaviours. Some mouse related gestures include:

  • onclick
  • ondblclick
  • ondrag
  • onmousedown
  • onmouseover

You can capture gestures applied to the window itself:

  • onload - triggered when the document completes loading. This is useful to trigger an initialization script that can then take control of the page. The initialization script is then guaranteed that the entire page has been parsed and the DOM is ready for manipulation.
  • onunload - triggered when you close navigate away from the page, you close the browser tab or quit the browser alltogether. This is useful to trigger scripts that clean up before leaving, or unsaved changes.
  • onresize - triggered when the browser window is resized. This is useful for triggering scripts that react to the size of the window and might do some complex page layout. This would complement (not replace) responsive layout that can already be achieved with CSS declarative rules.

Take a look at W3C's documentation and MDN's documentation for a thorough coverage of JavaScript event handling

JSON - JavaScript Object Notation


In JavaScript, data is represented using JSON or JavaScript Object Notation. JSON has become a very popular format for representing and sharing data in Web applications. JSON can be used to represent data to be displayed on the page, data collected from a form, data sent to/from a server, or the entire page itself. JSON is a simple notation for representing data of varying degrees of complexity combining JavaScript primitive data types, arrays, named properties, and nested data structures. Below is an example JSON object representing an employee: 

var alice = {
  "firstName" : "Alice",
  "lastName"  : "Wonderland",
  "ssn"       : "321-32-1234",
  "salary"    : 32132.12,
  "dob"       : new Date(1967,07,07)
};

The variable alice is set to a literal object that contains 5 properties: firstName, lastName, ssn, salary, and dob. Each of the properties has a literal value of different primitive types, e.g., firstName refers to the literal string Alice, the salary property refers to the number literal 32132.12, and the dob property has the literal date object instance.

The prorties in alice can be accessed by using the dot notation. For instance the code below gives alice a 10% raise: 

alice.salary = alice.salary * 1.1;

JavaScript Source Files

JavaScript can be declared and embedded within HTML documents, but it is usually a better idea to declare the scripts outside of the document and then load them separately by refering to them using the src attribute of the script element.

The browser makes separate HTTP requests for every resource referred to in the HTML document. For instance, images, CSS files, and JavaScript source files are loaded when referred to from the refering HTML document. Declaring resources outside of the HTML document increments their reusability. Various documents can use the same set of resources. The code below loads three common JavaScript source files used throughout the course. 

<script src="https://code.jquery.com/jquery-1.11.3.min.js"></script>
<script src="https://ajax.googleapis.com/ajax/libs/angularjs/1.4.5/angular.min.js"></script>
<script src="https://maxcdn.bootstrapcdn.com/bootstrap/3.3.5/js/bootstrap.min.js"></script>
<script src="js/index.js"></script>

The first three script elements load three common JavaScript libraries used throughout this course: jQuery, AngularJS, and Bootstrap. The last script, index.js, is downloaded from the same server where the HTML document is being downloaded. The remote scripts are loaded from remote servers in separate HTTP GET requests. These scripts could have been copied onto the local server and then downloaded from there instead of the remote server. Although this might seem safer, it actually is not. The servers from which they are downloaded from are special repositories called CDNs or Content Delivery Networks that are "guaranteed" to be available, perhaps even more available than our local servers, but if our server is not available, neither will the enclosing HTML document be available :) Scripts hosted in CDNs also have a performance advantage over local scripts. Since these are well known URLs, developers of various other sites tend to use them often as well. Browsers that have visited these other sites will cache them locally and not re-download them when they get to our site.

JavaScript Name Spaces

Downloading scripts from various sources and from different vendors is a very powerful feature that allows developers to quickly enhance Web site behaviours. There are although potential pitfals. When a JavaScript source is downloaded it is immediately executed. If a source file declares variables and functions, these are made available in the global namespace. They can be readily invoked from anywhere in other scripts. If we download scripts from various places, it is possible that different scripts declare variables or functions with the same name creating a name space collision. Scripts loaded last can potentially re-declare names declared by earlier scripts.

Namespace collition can be easily avoided by creating a namespace using a common JavaScript design pattern: Immediately-Invoked Function Expressions or IIFE. To create a namespace wrap your code within an IIFE as shown below: 

(function(){

  /* your code */

})();
