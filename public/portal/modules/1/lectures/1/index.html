Git

Git is a distributed version control system originally introduced in 2005 by Linus Torvalds, the same Linus who gave us Linux back in 1991. Git emphasizes speed and the ability to work distributedly. Instead of having a centralized code repository, every Git working directory is a full copy of a code repository with the whole complete history of code changes. Developers generally agree on one of these repositories as the master repositories from where they can all share the same code base.

Documentation

Git's official web site is https://git-scm.com . There you will find extensive documentation , books , and tutorial videos . This course will use Git as the source control software and will show the exact commands needed to use Git. We encourage you to visit Git's documentation for a thorough coverage of all the commands. Git will be used to deploy code to an online Web hosting service, to submit assignments, to retrieve example code, and to share source code in general.

Install a git client

There are several free git clients for the most operating systems. Some provide a graphical user interface and most of them also provide a command line interface. Regardless of what operating system you are using for this course we will be using the command line to interact with git. If you prefer to use the graphical user interface for your particular git client, please refer to the documentation on how to use the interface. The command line examples provided in this course are standard and will work the same regardless of your particular OS.

Github

Github.com is a service that allows hosting Git repositories. There are several tiers of service that range in price. The free version should be plenty for this course. It allows creating any number of public repositories with any number of developers collaborating on the same code repository. Being public anyone can see your code and clone it as their own and keep working on their own copy. They can't make changes to your repository unless invited to do so. Since it is open make sure not to include any sensitive information that might compromise your software systems such as usernames, emails, passwords, phone numbers, license keys, etc. If you want the repository to be private you'll need to pay for their private repository plans.

Steps:

  1. Create account in Github.com.
  2. Download Git client.
  3. Create a test repository in Github.com.
  4. Use terminal to set up local directory.
  5. Link local directory to remote repository.

Common Git Command:

  • git init | Create a new local repository
  • git status | Lists all new or modified files to be committed
  • git add | Add one or more files to staging (index)
  • git push | Send changes to the master branch of your remote repository
  • git commit | Commit changes to head (but not yet to the remote repository)
